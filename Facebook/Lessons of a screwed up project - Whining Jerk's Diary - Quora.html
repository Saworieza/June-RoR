Lessons of a screwed up project

I will be leaving off for a small vacation in few hours, but have to write this down.

The project

I was working on an insurance project through a consultancy in UK. They had too many other projects. One such project was selling videos. He asked me for a prototype that can be shown to the client and convince him to move from Drupal to RoR. I did the prototype, deployed in Digital ocean in a day, the client was impressed with the prototype which had a user flow that looked sleek.

Development

After making the prototype, I had no time to look into the project and it was handled by another team of my friend. I was trying to make my own app that I had told in my previous blog posts and  also involved in other project that I was already committed in.

After four weeks

I heard my friend's team was working on the project day and night to get the project ready. I joined them to add/fix parts related to the prototype feature I had done.
And I was told the client had shutdown the drupal site and wanted this site no matter what.

Screwed up production app

The site went alive. But the issues related to that, I would never forget in my entire life, I doubt if that kinda thing would ever happen
  • Production site goes down
  • Things go missing.
  • Orders from old site are no where to be seen.
  • Payment was not working...
  • It's enough,..
The consultancy person would forward  mails from customers to me. It was a shame to read those.

What went wrong?

This was an easy project to do, yet it was screwed up. So, blame? Mis - [communication, management, design, .....] Every miss resulted in the missings of the production app.

I saw the project was developed from the prototype I built in a day to impress the client. Now that was like a hackathon for me, and it should have been started from scratch.

For any software project, planning is more important. You plan for database design or things related to U/X experience or plans related to migration or plans related to choosing a tool to get your job done. Plan was not done.

The project's entire time was spent on fixing the problems caused by the design.

I knew the project was screwed up when I heard they were working day and night.


I sleep at least 8-9 hours a day. I believe for a job like programming, sleep is essential. When there is a pressure for me that I cannot handle, I would go to bed. One time, two, three guys wanted to fix their project/create a feature all immediately and I went straight to bed. After two hours of good sleep(nap?) fixed their issues.

Reason:? This thing is mental, when you are stressed, you are prone to make more mistakes, bugs that will reproduce exponentially everytime you do the same thing. And this project had that.

Miscommunication There were feature that was done but the client never asked. There were features that client wanted in x way but done in y way. I think it is because the communication was verbal, if it had been written, it would have been clear and the blame game could have been avoided.


The Technical Part

  • TDD or BDD - No
  • There should have been proper development flow tool. Tools like Git flow?
  • It was directly pushed into master, instead of separate branch, pull requests, code review.
  • Code smell.
  • Features were done to show the completion to the project managers(client) without thinking about the over all project and future issues.
  • There should have been a staging server before pushing changes directly to production server.

Finally, never give in to the pressure of client just coz they need to launch on a particular day or an auspicious day. If you feel the app is not ready, tell them, it would definitely save embarrasment that you should never face(even if the client decides to drop the project). This again comes under planning.

I underestimated this project, so were they. I thought this could be done in a week? I think estimation is the most important part in a development cycle. That should top the list.

Now, the client has moved the site to a different developer team based in UK. Now, months of man hours got wasted! reputation smacked. Gain? Lesson learnt.
