Most of you (who had used Rails) should be aware of the Ruby on Rails Tutorial: Learn Rails by Example twitter clone app. I spent some 15 days completing module by module and learn Rails. Later on i developed my own app that suited my need. But i always cherish the power of Rails, it makes you get things done in very small amount of time. I wanted to have a web dev Hackathon(though it din happen) and see how many apps people could develop in very small amount of time. I wanted this to boast! show off the faster development time of Rails :D

Let me check my speed (I am writing this side by side)

Now I am going to try my best to develop an app as fast i can. Hmm, I need an idea! Let me try a Facebook clone app.

What do i need?

0) A good layout something similar to Facebook.
1) User sign up, sign in, session management.
2) User relations.
3) Ability to create posts.
4) Ability to attach images to the posts.
5) Adding comments.
6) Liking posts/comments.
7) A feed

One of the important philosophies of Ruby on Rails is DRY(Do not repeat yourself). I am not going to write the same code(user management, comments, relations, ..) which i already did with my other apps. I am going to reuse these code from gems :)

Basic steps:
- Let me create a Rails project with the command:
    
1

rails new facebook



- I should remove the default index file:
    
1

rm public/index.html.erb

- I going to create a controller home with index action:
    
1

rails g controller home index

- Change root path in config/routes.rb
    
1

root to: 'home#index'

- Lemme check my app in the browser
    
1

rails s

- Yey! it works :D

0) A good layout (Bootstrap vs Zurb foundation)

I have been using Bootstrap for few of my projects. But its time to checkout zurb foundation.  Zurb follows mobile first approach which is kinda cool(have been hearing the fuss on responsive layout for long time :D) And it is SASS, so win for Rails app :)
Lets get into the Foundation Docs: Getting Started


- Installation.
There is a separate installation steps for Rails! Wow! Let me add the line gem 'zurb-foundation', '~> 4.0.0' to Gemfile and run
    
1

bundle install

Configure the app to use Foundation
    
1

rails g foundation:install


- I need the navbar for my site, the zurb docs is confusing, let me look at their source code. view-source:http://foundation.zurb.com/docs/ I could see their navbar under nav tag :) I am going to copy the code and make my navbar with small modifications. I am copying this code into _header partial :) and including it in the layouts.

- I should change the default black color of the navbar. I should find a suitable color. Color Hex Color Codes helps in choosing few colors. I have chosen a color, let me check the comments inside foundation_and_override.scss file. Background color of topbar - got it, changed the value.

- Yey! this works.

1) User related stuff.


I am tired of writing the logic from scratch and lemme see a gem. The most popular gem out there would be Devise.

- Go to github page of devise for instructions devise to follow installation, and others. This is fairly simple, let me check it out.

- Installation
    
1

gem 'devise' # add this to Gemfile and bundle install

- Configuration
    
1

rails g devise:install #this tells few things like adding lines to application.html.erb and application.rb

- Generate user model
    
1

rails g devise User

- Run migrations for user
    
1

rake db:migrate

- Restart server and check the url Page on Localhost:3000 . Wow! it works :D

I need to do few edits(css like button) in the files.

I am going to redirect home page to users/sign_in if the user is not signed in.  Need to add some code in index action of home controller.


Now step1 is complete :)


2)  User relations.


I did a Google search for "rails gem follow" and got socialization and act_as_follower gems github pages. Now looking at the github page, socialization is more recent and has considerable number of stars/forks. So, let me try socialization gem. socialization

Hey! looks like this gem can even complete the step6 of liking posts/comments :D
- Installation
    
1

gem "socialization" #bundle install

- configuration
    
1

rails generate socialization -s

#this is weird :D

Hmm, i should add "follow button" to user profile! Oh! i don't have a user profile.
    
1

rails g controller users index show

# only index, show(profile) would be useful, others are already being done by devise


Now I need to edit the show file of users! Just like how it is in Facebook profile.

Now i have added a follow form with follow buttons(with ajax calls) something similar to Ruby on Rails Tutorial: Learn Rails by Example
I see I have not added a column "name" for users. Lemme add it now.
    
1
2

rails g migration add_name_to_users name:string
	rake db:migrate

Need to insert it into attr_accessible of user model and validate it with
    
1

validates :name, presence: true



3) Ability to create posts.


This must be simple. Lemme try the classic scaffold command :D
    
1
2

rails g scaffold posts user_id:integer content:text
	rake db:migrate


Inside the posts/form, the user field should be removed as it should be related while posting.
Now have to display _form.html.erb(posts) partial inside the home page. 

    
1

<%= render 'posts/form' %>

Also this form involves @post so a line should be added to index action
    
1

@post = Post.new


And relation between post and user should be established.
In models/user.rb
    
1

has_many :posts

In models/post.rb
    
1

belongs_to :user


Now the user can create posts, yey! To see the list of posts, user needs to go to /posts. We will take care of making a feed later once we complete other modules.


4) Attach images to the posts/users


A simple Google would give out the answer. Paperclip gem :D paperclip This shows before installing paperclip, imagemagick must be installed and corresponding path must be set. As usual adding to Gemfile and bundle install.

Now images must be needed to both users and posts. I followed the instructions as in the github page. Full copy pasting(model and views), but a small change to the default url to point to the default image :)

    
1
2
3

rails g paperclip user avatar	
	rails g paperclip post avatar
	rake db:migrate


Now i have to insert the image in the profile of the user with some css to adjust the size. Update edit/new part of user/post creation files

Wow! now it works :D



5) Adding comments


As usual proceed to Google for 'rails gem comments'. Hmm, getting suggestions on 'opinio' and 'act_as_commentable' gems. By the same previous logic based on the github page, I am going to choose act_as_commentable gem :D

Gem is even made for Rails 4, Cool :) And the acts_as_commentable instructions are simple with less craps.

After trying this gem, it only adds to confusion and does not simplify anything. I would rather create the comments with simple steps.
    
1
2
3

rails g model comments content:text post:belongs_to user:belongs_to
    rails g controller comments
    rake db:migrate


A small comment form which should be shown in the posts' show page

Way to go :) Some times gems are not best :P A basic commenting system is done.

6) Liking posts/comments

Already in step 2) socialization gem created models for likes and mentions. Since i am resuing the gem, i copying the view files, actions created for it :D
Crap, wasted few minutes with the careless error "act_as_likeable" instead of "acts_as_likeable". Now, since like is for both comments/posts, i have to specific the type along with id when liking or unliking. Should be easy to get the type with .class method. There was not much a problem in implementing likes, I should say "socialization" is quite a good gem :)

7) Feed.


Now coming to the final part, implementing feed.

Oh shit! got into some ridiculous argument in Facebook now. Where were we ? Ah! Feed! ya.

Hmm, lemme do the Google stuff! :D

  I again got into Facebook now. IF you ever want to do something fruitful, first close the Facebook tab and after reading the post close the Quora tab too :P

Okay, the top results of Google has the  most recent Railscasts episode #406 Public Activity . When you have a railscasts episode why worry :D

So, he has used pubic_activity gem. Should make things easier ;)

 When installing this gem, got some notice on the latest version on public activity. Hmm, so it would have been made better after railscasts episode? Okay, lemme check the github page public_activity


Wow! this github page is much more clearer :D I am going to follow this :)
    
1

rails g public_activity:migration


I tried tracking the posts, so far it looks ridiculously awesome. Need to make it look better and should implement across models.  I am going to sleep, have been doing too many things today.

Yeah! lets start again :) Hmm, the public activity tutorial is really an easy one :) Let me try to get the activity of all the people I follow. But socialization github READme does not have a following(number). Looking at the issues page, there is a followees method, yey! :D
        
1
2
3
4

followees_ids = current_user.followees(User).pluck(:id)
        #get only the ids of the people current_user folllows
        followees_ids << current_user.id
        @activities = PublicActivity::Activity.where(owner_id: followees_ids, owner_type: "User")


This is not showing any activity! What's wrong? I am going to fire up rails console
    
1
2

rails c
    PublicActivity::Activity.all


Ah! this shows the owner id is nil, lemme check Railscasts episode! Whoa! the episode has something to set the owner id while tracking. The github page of public activity should have this in their README :/

After adding the lines of owner in the tracking model and Application controller as per Railscasts, it works!!

Now i should track the liking of people, it is one of the significant features in Facebook and in Quora(upvotes), these always let people discover cool contents. After implementing posts activity, like activity was not much of an issue, lemme track the comment activity now which should be exactly similar to like activity. whoah! it was indeed super simple.

Now i have made a very basic Facebook clone :) Did not make stats in Profile page, its not much of an issue and have not done many css. So, a very basic Facebook app is ready :)
